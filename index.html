What This Project Does

This project is a Digital Twin of a physical ATM system[cite: 111]. Instead of a simple script, it uses advanced Object-Oriented Programming (OOP) to model real-world banking entities[cite: 6, 7].

  • The Data Vault: Securely manages account balances using private encapsulation[cite: 145, 217].
  • Validation Gate: Robustly filters user input to prevent system crashes[cite: 146, 173].
  • Transaction Engine: Follows the Input-Process-Output (IPO) model for reliability[cite: 149, 153].

How To Run It

To run this project locally, ensure you have the Java Development Kit (JDK) installed.

  1. Clone this repository to your machine.
  2. Navigate to the src/ directory.
  3. Compile the project:
    javac main/Main.java
  4. Execute the program:
    java main.Main

Core Implementation (The Vault)

Following the Separation of Concerns principle, the logic is kept entirely separate from the UI[cite: 241, 254].

// BankAccount Class Snippet
public class BankAccount {
    private double balance; // Secure Encapsulation

    public boolean withdraw(double amount) {
        // Business Rule: Validate sufficient funds
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false; // Reject Transaction
    }
}
